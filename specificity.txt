Paragraph A: It gets the p styles only. So the browser uses gray text, 1rem font size, Georgia, line-height: 1.4, and padding: 4px 0.
Paragraph B: It gets styles from p and .info, but the class wins where both set the same property. So it uses steelblue text, 1.25rem, Arial, light blue background, 8px 14px padding, and the left border from .info. line-height still comes from p.
Paragraph C: It gets styles from p and #warning, and the id wins over the element selector. So it uses dark red text, 0.8rem, italic, uppercase text, wider letter spacing, dashed border, pink background, Courier New, and 10px padding. line-height still comes from p.
Paragraph D: It matches all four rules, but the most specific one is #warning.info, so that rule wins for the properties it defines. Color becomes dark orange because #warning.info beats #warning and .info. Font-family is Courier New because only #warning sets it. For properties not set by the more specific rule, the browser keeps the value from the next best rule, like padding: 10px from #warning and line-height: 1.4 from p.
For the h1 example: the external style.css gives h1 a color, italic font style, and padding: 10px. When an internal h1 rule is added in the <style> tag with a different color, that color wins because both rules have the same selector strength and the internal rule comes later. The h1 keeps the other properties from the external file unless the internal rule changes them too.
